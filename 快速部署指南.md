# 快速部署到 GitHub Pages

## 步骤 1: 在 GitHub 创建仓库

1. 登录 GitHub (https://github.com)
2. 点击右上角 **"+"** → **"New repository"**
3. 填写信息：
   - **Repository name**: `wanbei-knowledge-graph` (或你喜欢的名字)
   - **Description**: 皖北军事环境成因知识图谱平台
   - **Visibility**: 选择 **Public** (GitHub Pages 免费版需要公开仓库)
   - **不要**勾选 "Initialize this repository with a README"
4. 点击 **"Create repository"**

## 步骤 2: 复制仓库地址

创建完成后，GitHub 会显示仓库地址，类似：
```
https://github.com/YOUR_USERNAME/wanbei-knowledge-graph.git
```
**复制这个地址**，下一步会用到。

## 步骤 3: 运行部署脚本

### 方法 A: 使用批处理脚本（Windows）

1. 双击运行 `deploy-to-github.bat`
2. 按提示操作：
   - 输入提交信息（直接回车使用默认值）
   - **粘贴完整的 GitHub 仓库地址**（例如：`https://github.com/username/repo-name.git`）

### 方法 B: 手动执行命令

如果脚本有问题，可以手动执行：

```bash
# 1. 进入项目目录
cd 展示网站

# 2. 初始化 Git（如果还没有）
git init

# 3. 添加所有文件
git add .

# 4. 提交
git commit -m "Initial commit"

# 5. 添加远程仓库（替换为你的实际地址）
git remote add origin https://github.com/YOUR_USERNAME/YOUR_REPO_NAME.git

# 6. 推送到 GitHub
git branch -M main
git push -u origin main
```

**注意**: 如果是第一次使用 Git，可能需要先配置：
```bash
git config --global user.name "Your Name"
git config --global user.email "your.email@example.com"
```

## 步骤 4: 启用 GitHub Pages

1. 在 GitHub 仓库页面，点击 **"Settings"**（设置）
2. 在左侧菜单找到 **"Pages"**
3. 在 **"Source"** 部分：
   - 选择 **"Deploy from a branch"**
   - **Branch** 选择 **"main"**
   - **Folder** 选择 **"/ (root)"**
4. 点击 **"Save"**

## 步骤 5: 等待部署完成

- GitHub Pages 通常需要 **1-2 分钟** 完成部署
- 部署完成后，访问地址：
  ```
  https://YOUR_USERNAME.github.io/YOUR_REPO_NAME/
  ```
  例如：`https://zhangsan.github.io/wanbei-knowledge-graph/`

## 常见问题

### Q1: 提示 "repository doesn't exist"
**A**: 确保仓库地址格式正确，必须是完整的 URL：
- ✅ 正确：`https://github.com/username/repo-name.git`
- ❌ 错误：`wanbei-knowledge-graph` 或 `username/repo-name`

### Q2: 提示 "Authentication failed"
**A**: 需要配置 Git 用户信息：
```bash
git config --global user.name "Your Name"
git config --global user.email "your.email@example.com"
```

### Q3: 提示 "Permission denied"
**A**: 
- 确保仓库是 Public，或者你的账户有 GitHub Pages 权限
- 检查仓库地址是否正确
- 可能需要使用 Personal Access Token（如果启用了双因素认证）

### Q4: 页面显示 404
**A**: 
- 检查 GitHub Pages 设置是否正确
- 确保 `index.html` 文件在根目录
- 等待几分钟后刷新页面

### Q5: 资源文件加载失败
**A**: 
- 检查所有 JSON、GeoJSON 文件是否都已提交
- 查看浏览器控制台的错误信息
- 确保文件路径使用相对路径（代码中已正确配置）

## 更新网站

修改代码后，只需：

```bash
git add .
git commit -m "更新描述"
git push
```

GitHub Pages 会自动重新部署。
